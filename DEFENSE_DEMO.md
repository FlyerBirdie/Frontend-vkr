# Демонстрация на защите (краткий сценарий)

## Предусловия

- Настроены учётные данные планировщика и подключение к базе по инструкции репозитория (файлы окружения в каталоге backend).
- Запущены сервер приложения (backend) и режим разработки фронта (`npm run dev` в каталоге frontend).
- Выполнен **вход** на странице логина теми же учётными данными, что выданы для планировщика — иначе защищённые разделы недоступны.

## Ручной чеклист auth (4 пункта)

1. **Без токена:** в новой вкладке/после очистки данных сессии браузера открыть защищённый раздел (например заказы) — ожидается перенаправление на страницу входа.
2. **Неверный пароль:** отправить форму входа с ошибкой в пароле — сообщение об ошибке, доступ не сохраняется.
3. **Успешный вход:** корректные логин и пароль — переход в приложение; в инструментах разработчика видно сохранённый маркер сеанса.
4. **Защищённые запросы:** во вкладке «Сеть» у запросов к данным после входа есть заголовок авторизации; при истечении срока или недействительном маркере — возврат к странице входа.

## Автотесты и скрипты (без обхода auth)

- Отдельного сквозного e2e в репозитории **нет** — ручная проверка по чеклисту выше.
- `Frontend`: `npm test` — только unit-тесты хелперов и разметки, **без** сетевых вызовов к серверу.
- `Backend`: `pytest` — проверки авторизации и доступа к данным по сценариям из тестов репозитория.

---

**Сценарий демонстрации функционала** (после входа):

1. В навигации открыть **Рабочие** и **Оборудование** — показать справочники; при необходимости добавить запись кнопкой «Добавить».
2. Открыть **Техпроцессы** → перейти в деталь ТП — показать список задач (последовательность, длительность, профессия, модель оборудования); при необходимости скорректировать или добавить задачу.
3. Открыть **Заказы** — создать новый заказ или отредактировать существующий: прибыль, привязка к ТП, плановое окно заказа (даты вводятся в том же часовом поясе, что и на странице «Расписание»).
4. Перейти в **Расписание**: задать **плановый период** → **Запустить планирование**; дождаться блока «Итог планирования» (период, прибыль, вкл./искл., операции).
5. Переключить вид **Гантт (ресурсы)** — временная диаграмма по выбранной группировке (рабочие / оборудование).
6. Переключить **План по заказам** — те же операции в привязке к заказам.
7. Вкладка **Операции** — сохранённые операции, поиск по названию заказа, постраничный просмотр.
8. Открыть **Отчёт** в шапке (или кнопку «Открыть отчёт» на странице расписания) — метрики загрузки и сводные блоки; если в сводке есть **исключённые заказы**, вернуть внимание к таблице «Исключённые заказы» в «Итог планирования» на **Расписание** (частичное включение недопустимо — заказ целиком в расписании или в списке исключённых с причиной).
