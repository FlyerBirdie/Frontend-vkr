# Фронтенд оперативного планирования производства

Веб-интерфейс для расчёта расписания, справочников (заказы, рабочие, оборудование, техпроцессы) и аналитики загрузки ресурсов. Репозиторий рассчитан на публикацию **исходников на GitHub** без секретов и без артефактов сборки.

## Стек

- **Vite 5** + **React 18** + **TypeScript**
- **react-router-dom** (маршрутизация)
- **Tailwind CSS** (стили)
- **Recharts** (графики на странице аналитики)
- **Vitest** + Testing Library (юнит-тесты)

## Требования

- **Node.js** актуальной **LTS**-ветки (рекомендуется **20.x** или **22.x**), вместе с **npm** из комплекта Node.

Проверка версии:

```bash
node -v
npm -v
```

## Установка и команды

| Команда | Назначение |
|--------|------------|
| `npm install` | Установка зависимостей |
| `npm run dev` | Dev-сервер Vite (по умолчанию [http://localhost:5173](http://localhost:5173)) |
| `npm run build` | Проверка TypeScript (`tsc -b`) и production-сборка в каталог `dist/` |
| `npm run preview` | Локальный просмотр собранного приложения (после `npm run build`) |
| `npm test` | Запуск тестов (Vitest) |

## API в режиме разработки

Запросы к backend идут на префикс **`/api`** (см. `src/api.ts`, константа `API_BASE`).

В **`vite.config.ts`** настроен прокси: запросы с dev-сервера на **`/api`** перенаправляются на **`http://localhost:8000`**. То есть backend должен быть доступен на порту **8000**, а фронт — открывать через Vite (`npm run dev`). Отдельная настройка CORS в браузере для этого сценария не нужна: браузер обращается к тому же origin, что и страница (Vite), а прокси ходит на backend уже с сервера разработки.

## Часовой пояс

Планирование, период на странице **«Расписание»** и плановые окна **заказов** вводятся и показываются как **Europe/Samara (UTC+4)**; в JSON для API уходит timezone-aware ISO (обычно UTC с `Z`). Реализация: `src/samaraTime.ts`, преобразование полей `datetime-local` — `datetimeLocalToIsoUtcFromSamara` в `src/planningPeriod.ts`.

## Секреты и `.env`

- **Не коммить** файлы `.env`, `.env.local` и подобные, если в них окажутся пароли, ключи API или другие секреты. Они перечислены в **`.gitignore`**.
- В коде приложения **нет пользовательских переменных `VITE_*`**: используется только встроенный у Vite `import.meta.env.BASE_URL`. Отдельный **`.env.example`** в репозитории не добавлен. Если позже появятся переменные с префиксом **`VITE_`**, имеет смысл завести **`.env.example`** с пустыми/фиктивными значениями и описать их здесь в README.


## Production-сборка

После `npm run build` статика лежит в **`dist/`**. Раздача — через любой статический хостинг или reverse-proxy; префикс **`/api`** в production нужно настроить на ваш backend отдельно (прокси на уровне nginx/Caddy и т.п.) — dev-прокси Vite в собранном приложении не используется.
